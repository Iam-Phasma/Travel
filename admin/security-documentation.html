Security Documentation

This page summarizes active protections in plain language. It is intentionally written without implementation details so the team can understand safeguards without exposing technical internals.

Overview All Signed-In Users Standard Users Admin Users Super Admin Users

All Signed-In Users

These controls apply broadly to authenticated activity and help reduce risk from stale sessions or unauthorized continued access.

Inactivity Session Timeout

Idle sessions are ended after a warning so unattended devices do not remain signed in.

Access Revocation Enforcement

If an account is disabled, access is blocked and active sessions can be ended promptly.

Server-Validated Permissions

Critical permissions are verified on trusted backend paths, not only in browser UI checks.

Standard Users

These protections focus on safe sign-in and account recovery, while reducing automated abuse attempts.

CAPTCHA Verification

Human verification is required on login and password reset to reduce bot traffic.

Failed Login Cooldowns

Repeated incorrect logins trigger temporary waiting periods to slow password guessing.

Network Abuse Throttling

Excessive failed attempts from the same network are temporarily limited.

Password Reset Request Limits

Reset requests are rate-limited to reduce spam and targeted reset abuse.

Neutral Recovery Messaging

Recovery responses avoid revealing whether a specific email is registered.

Password Strength Requirements

New passwords must pass baseline strength and predictability checks.

Admin Users

Admin-level access has additional controls to lower risk in privileged sessions.

Single Active Session Guard

Privileged accounts are protected from parallel active sessions across multiple devices.

Recurring Session Revalidation

Admin sessions are periodically rechecked to detect invalid or outdated access.

Admin Route Protection

Admin pages validate role eligibility before privileged panels or actions are available.

Super Admin Users

Super-only controls protect high-impact account and role operations.

Restricted User-Management Actions

Role changes and access toggles are limited to super admin authorization paths.

Last-Super Safeguards

Safety checks prevent disabling or downgrading the final active super admin account.

Session Validation On Sensitive RPC Calls

High-impact admin operations verify active privileged session context before applying changes.

Controlled Maintenance Tools

Risky maintenance operations are exposed only in restricted super-admin contexts.
